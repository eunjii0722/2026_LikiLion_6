## 프로젝트 개요

**(서비스명) AI 자동화 웹앱**은 업무 자동화를 쉽게 설계하고 시뮬레이션할 수 있는 웹 기반 프로토타입입니다.
사용자가 직관적인 인터페이스를 통해 워크플로우를 구성하고, AI 기반 자동화 테스트를 수행할 수 있습니다.

***시연할 핵심 시나리오 “카카오톡으로 수강 신청이 오면, 구글시트에 자동으로 정리하고 확인 메시지를 보내줘.”***

---

## 프로토타입 구성

이 프로젝트는 UX/UI 설계 프로세스의 여러 단계를 포함합니다:

### 1️⃣ **IA (Information Architecture)**
- 서비스의 전체 메뉴 구조 및 정보 계층 정의
- 사용자 여정(User Journey) 맵핑
- 각 기능 간 상호작용 관계도

### 2️⃣ **와이어프레임 (Wireframe)**
- 각 화면별 정보 배치의 초안
- 기능 배치 및 레이아웃 검토
- 인터랙션 플로우의 기본 설계

### 3️⃣ **Hi-Fi 프로토타입 (High-Fidelity)**
- Figma에서 디자인한 완성도 높은 화면
- 실제 색상, 타이포그래피, 스타일 적용
- 상세한 비주얼 디자인

### 4️⃣ **클릭형 프로토타입 (Interactive Prototype)** ⭐
- **본 프로젝트**: React 기반의 상호작용 가능한 프로토타입
- 버튼 클릭으로 화면 이동 시뮬레이션 가능
- 발표 중 실제 동작하는 앱처럼 시연 가능
- 실제 사용자 피드백 수집 가능

### 5️⃣ **시연 시나리오 (Demo Script)**
- 발표자가 따라할 수 있는 단계별 시연 순서
- 각 화면에서 강조할 핵심 기능 설명
- 예상 질문과 답변

## IA (정보 아키텍처)

### 전체 서비스 메뉴 구조

```
(서비스명)
├── 메인 화면 (Home)
│   └── 프로젝트 목록 및 소개
├── 입력 화면 (Input)
│   └── 자동화 규칙/워크플로우 입력
├── 분석 화면 (Analysis)
│   └── 입력된 워크플로우 분석 및 검증
├── 워크플로우 화면 (Workflow)
│   └── 워크플로우 상세 정보 및 설정
└── 결과 화면 (Result)
    └── 테스트 결과 및 통계
```

## 화면 구조

### 메인 플로우 (일반 사용자 플로우)

| 화면 | 설명 | 주요 기능 |
|------|------|---------|
| **Home** 🏠 | 메인 랜딩 페이지 | 프로젝트 소개, 시작 버튼 |
| **Input** ✍️ | 워크플로우 입력 | 자동화 규칙 입력, 조건 설정 |
| **Analysis** 🔍 | 워크플로우 분석 | 입력값 검증, AI 분석 결과 |
| **Workflow** 🔄 | 워크플로우 상세 | 플로우 구조 시각화, 세부 설정 |
| **Result** 📊 | 테스트 결과 | 실행 결과 통계, 성공/실패 분석 |

### 데모 플로우 (발표용 시연 플로우)

| 화면 | 설명 | 주요 기능 |
|------|------|---------|
| **Demo Standby** ⏸️ | 데모 시작 대기 | 데모 초기화, 시연 준비 |
| **Demo Message** 💬 | 메시지 처리 | 메시지 수신 및 분석 시뮬레이션 |
| **Demo Extract** 📋 | 정보 추출 | 메시지에서 핵심 정보 추출 |
| **Demo Save** 💾 | 데이터 저장 | 추출된 정보 저장 완료 |
| **Demo Complete** ✅ | 완료 화면 | 처리 완료 확인, 결과 요약 |

## 시연 시나리오

### 발표 스크립트 가이드

#### 1단계: Home 화면에서 시작
```
"안녕하세요, (서비스명) 프로토타입 발표를 시작하겠습니다.
이것은 업무 자동화를 쉽게 설계할 수 있는 웹 앱입니다.
[홈 버튼 클릭] 버튼을 눌러서 플로우를 시작해보겠습니다."
```

#### 2단계: Input 화면 - 워크플로우 입력
```
"먼저 자동화하려는 업무를 입력합니다.
여기서 자동화 규칙과 조건을 설정할 수 있습니다.
[입력 필드 데모] 예를 들어, '고객 요청이 들어오면 자동으로 이메일 발송' 같은 규칙을 입력합니다."
```

#### 3단계: Analysis 화면 - AI 분석
```
"입력한 내용을 AI가 분석합니다.
[분석 버튼 클릭] 규칙의 타당성과 실행 가능성을 검토합니다.
결과적으로 이러한 최적화 사항들을 제안합니다."
```

#### 4단계: Workflow 화면 - 시각화
```
"분석된 결과를 워크플로우로 시각화합니다.
각 단계가 어떻게 연결되는지 한눈에 볼 수 있습니다."
```

#### 5단계: Result 화면 - 최종 결과
```
"마지막으로 시뮬레이션 실행 결과를 보여줍니다.
성공률, 처리 시간, 오류 분석 등의 통계가 표시됩니다."
```

#### 보너스: Demo 플로우
```
"만약 실제 동작을 보고 싶으시면, 데모 모드에서 실제 시나리오를 시연할 수 있습니다.
[Demo 링크] 여기서 메시지 수신부터 저장까지의 전체 흐름을 보여드립니다."
```

### 예상 질문과 답변

**Q1: 실제 업무에 어떻게 적용되나요?**
```
A: 각 조직의 특성에 맞게 커스터마이징할 수 있습니다. 
API 연동을 통해 실제 시스템과 연결할 수 있으며, 
클라우드 환경에서 자동 실행됩니다.
```

**Q2: 여러 개의 워크플로우를 동시에 관리할 수 있나요?**
```
A: 네, 홈 화면의 프로젝트 목록에서 여러 워크플로우를 동시에 관리하고, 
각각에 대해 독립적으로 테스트 및 배포할 수 있습니다.
```

**Q3: 오류 발생 시 어떻게 처리되나요?**
```
A: Result 화면에서 상세한 오류 분석과 로그를 제공합니다. 
각 단계별 성공/실패 여부를 추적할 수 있으며, 
필요시 워크플로우를 수정하여 재실행할 수 있습니다.
```

## 실행 방법

### 필수 사항
- **Node.js** 18.0 이상
- **npm** 패키지 매니저

### 설치 및 실행

#### 1. 의존성 설치
```bash
npm i
```

#### 2. 개발 서버 실행
```bash
npm run dev
```

#### 3. 브라우저에서 접속
```
http://localhost:5173
```

자동으로 브라우저가 열리지 않으면, 위 주소를 직접 입력합니다.

### 종료

```bash
Ctrl + c
```

## 개발 환경

### 사용 기술 스택

- **Frontend Framework**: React 18+
- **Routing**: React Router
- **UI Components**: Radix UI
- **Icons**: Lucide React, Material-UI Icons
- **Styling**: Tailwind CSS
- **Build Tool**: Vite
- **CSS-in-JS**: Emotion
- **Animation**: Motion
- **Utilities**: clsx, class-variance-authority

### 프로젝트 구조

```
src/
├── main.tsx                 # 애플리케이션 진입점
├── app/
│   ├── App.tsx             # 루트 컴포넌트
│   ├── routes.tsx          # 라우팅 설정
│   └── components/
│       ├── Layout.tsx           # 메인 레이아웃
│       ├── HomeScreen.tsx       # 홈 화면
│       ├── InputScreen.tsx      # 입력 화면
│       ├── AnalysisScreen.tsx   # 분석 화면
│       ├── WorkflowScreen.tsx   # 워크플로우 화면
│       ├── TestResultScreen.tsx # 결과 화면
│       ├── demo/                # 데모 플로우
│       │   ├── DemoLayout.tsx
│       │   ├── DemoStandbyScreen.tsx
│       │   ├── DemoMessageScreen.tsx
│       │   ├── DemoExtractScreen.tsx
│       │   ├── DemoSaveScreen.tsx
│       │   └── DemoCompleteScreen.tsx
│       ├── figma/               # Figma 연동 컴포넌트
│       └── ui/                  # 재사용 가능한 UI 컴포넌트
└── styles/                  # 글로벌 스타일
```
